# Expense Tracker CLI

Build a command-line expense tracker in TypeScript from scratch.

In Task 1 you converted an existing JavaScript app to TypeScript. This time there is no starter code — you design the architecture and write everything yourself. The only things provided are the project setup files (`package.json`, `tsconfig.json`).

## Getting Started

```bash
npm install
```

A starter `src/main.ts` is included so you can verify the setup works:

```bash
npm start
```

The `start` script runs `node src/main.ts`. Node.js can execute TypeScript directly — no build step needed.

> [!IMPORTANT]
> When importing between your `.ts` files, use the `.ts` extension in the import path:
> ```ts
> import { Expense } from "./types.ts";
> ```

## What to Build

An interactive CLI app that lets the user:

1. **Add an expense** — with an amount, category, date, and description
2. **List all expenses** — in a formatted table
3. **Delete an expense**
4. **Exit**

When all of the above works, extend the app with:

5. **Filter expenses** — by category and/or date range
6. **View a category summary** — total and count per category
7. **View a monthly summary** — total and count per month

Features 5–7 are more challenging. It's okay if you don't get to all of them.

### Expense Properties

Each expense stored in the app has the following properties:

- **id** — a unique identifier, generated by the app (not entered by the user)
- **amount** — a number
- **category** — one of the fixed categories listed below
- **date** — a string in `YYYY-MM-DD` format
- **description** — a string

Expenses should be persisted to `data/expenses.json` so they survive across runs.

### Categories

Use a fixed set of categories: Food, Transport, Housing, Entertainment, Utilities, Health, Other.

## Suggested Build Order

Don't try to build everything at once. Work through these stages, testing as you go:

1. **Start with your core types** — create an `Expense` interface and a type for categories. Don't try to define every type up front — add more types later as the need arises.
2. **Build file persistence** — write functions (or a class) to read and write JSON to `data/expenses.json`. Handle the case where the file or directory doesn't exist yet (create the directory programmatically, return an empty array for a missing file).
3. **Implement add and list** — get adding and listing expenses working first. This proves your types, persistence, and basic flow all work together.
4. **Build the menu loop** — use `prompt-sync` (already installed) to create an interactive menu. You've used this package before — the pattern is the same.
5. **Add delete** — now that CRUD basics work, add the ability to delete by ID.
6. **Add formatting** — make the output look nice with aligned columns.
7. **Stretch: filter and summaries** — if you have time, add filtering by category/date range and the summary views.

## TypeScript Concepts to Practice

This assignment is your chance to use the TypeScript features from the learning material. Try to incorporate as many as you can:

- **Type annotations** — on function parameters, return types, and variables
- **Interfaces** — to define the shape of an expense, filters, summaries
- **Type aliases and unions** — e.g. a string literal union for the fixed set of categories
- **Utility types** — e.g. `Omit` to derive a "new expense" type (everything except `id`) from your expense interface
- **Optional properties** — for filter fields that the user may or may not provide
- **Generics** — e.g. a reusable JSON file storage class that works with any data type
- **Type guards / narrowing** — e.g. checking user input before using it

Don't worry if you don't use every single concept. Focus on writing clean, working TypeScript.

## Suggested Architecture

You're free to structure your code however you like. One approach that works well:

```
src/
  types.ts              — interfaces, type aliases, and union types
  storage.ts            — reads/writes a JSON file (generic)
  expense-manager.ts    — business logic: add, delete, filter, summarise
  formatters.ts         — functions that turn data into formatted strings
  main.ts               — the interactive menu loop
```

### Hints

- Use `prompt-sync` for user input (already installed):
  ```ts
  import promptSync from "prompt-sync";
  const prompt = promptSync();
  const name = prompt("Enter your name: ");
  ```
- Use `node:fs` (`readFileSync`, `writeFileSync`, `mkdirSync`, `existsSync`) for file persistence. Create the `data/` directory programmatically if it doesn't exist — don't assume it's already there.
- Use `crypto.randomUUID()` to generate unique IDs for expenses.
- ISO date strings (e.g. `"2026-01-15"`) can be compared with `<` and `>` as regular strings.
- Use `.padEnd()` and `.padStart()` to align columns in your output.
- Use `toLocaleString("nl-NL", { style: "currency", currency: "EUR" })` for Euro formatting.
- Use `toLocaleDateString("nl-NL", { ... })` for Dutch date formatting.
